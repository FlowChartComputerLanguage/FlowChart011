Chapter 1  Introduction

 

FlowChart Computer Language is an attempt to be able to program in any computer language via the use of visual FlowCharting.  Actually it's more accurate to call it a software schematic.  A FlowChart is usuallty thought of as the logical flow of a program.  A Data Flow Diagram is a program method where each  'symbol' works when all of the required data arrives to it.  A software schematic has both logic and data flow.  This FlowChart program works as a software schematic, a Data Flow Diagram, and as a traditional FlowChart.

 

FlowChart is made to be used by anyone, in any language. It is designed to be used with graphics so that it is not dependent on text based programming.  This means that it can be used by anyone whether or not they know the letters used by most European languages.  FlowChart is intended to be used without language.  As people use this program and come up with ideas to refine it, it will become more language independent.

 

FlowChart can be used for a number reasons. It allows the source code of a program to be documented in a form that is easy for anyone to understand.  It is made to adapt to any new computer language that has logic and data flow.  It is not intended for a specific spoken, written, or programming language. People who speak English should be able to use the process as easily as someone who speaks or writes in another language.

 

FlowChart is a picture oriented representation of what the source code of a computer program is doing. It is therefore easier to understand providing that the symbol designs makes sense to them.  There is an assumption that certain symbols have common universal meaning.  The stop sign, a circle with an X through it, a stick figure of a man, or a woman, these all have universal meaning.  So instead of words, there are picture symbols that represent the operations of the steps that a computer program is doing. 

 

The text of each symbol of FlowChart can be changed to how it uses the logic and data flow.  Changing the text of a symbol will change it.  So symbols can be changed into another programming language, or the dialect of the compiler.  This enables anyone who understands a FlowChart to be able to understand what the program is doing, even if they do not know the computer language that it is using.  

 

The FlowChart program translates from or to almost any programming language.  Of course, like anything new and complex, these first versions will have lots of bugs.  With the hundreds of dialects of just the BASIC computer language, it would be impossible for me to code and test for every one.  So I am relying on others to help test other languages, finding bugs, and add new ideas.

 

The symbols that make up a flow chart can be customized to any computer language.  Conversion between computer languages can be made on a line to line conversion. This should make it easier to translate any computer language into another computer language by changing each of the symbol's text that creates the computer program source code.  This requires that the programming language has a flow of logic, uses variable names, and has a keyword syntax. It will not take into consideration the inherent advantages or pitfalls of each computer language.

 

One of the goals of FlowChart is to have a universal symbol language that can be compiled into any computer language.  Another goal is to be able to have programmers who can create code in any computer language without having to know the syntax and detail requirements of a specific computer language.

 


The FlowChart has two basic units.

 

FlowChart Screen:

 

 

The FlowChart screen shows what a program does. It shows the flow of logic and data from one symbol to another.  This represents what a computer program is doing, how the data is moved around, tested, and performs computations. The symbols can be as simple as a single machine instruction, or as complicated as a batch file that runs multiple programs. 

 

FlowChart is made up of three basic items:

  • Paths showing colored and styled lines that connect the symbol points together. They show the flow of logic, and data.

  • Symbols show each step of a program. 

  • Constants will connect paths with data.

 

 

Symbol Screen:

 

 

 

Each symbol is made up of a number of parts:

  • unique name

  • colored lines

  • points where paths connect, each point has:

    • a name

    • a data type such as an integer, a real number, a character string, a record structure, an array, and any other data type of a specific compiler

    • either an input, output, or both

    • optional or required connected to a path

  • syntax code that is used to create the actual program source code using the point name   

  • description

  • file name

  • version/author

  • an example of the program code

Chapter 3   Symbols

All programs must have a symbol named, 'Start' which shows where the program begins.  Even subroutines and functions must have a start.  Device drivers will have multiple entry points coming out of the start.  

 

A FlowChart is expanded by placing more symbols on the FlowChart screen. Each symbol added will define an instruction of what the program is does.

 

The diagram will be completed by adding symbols and paths between the dots of symbols.  Make sure that the points have matching colors.  The color and size of the dots on symbols show the datatype being used by each symbol.

 

    After the FlowChart is complete then save the FlowChart.

     

    Next step is to generate and save the source code.  If there are no errors, then the program is for whatever compiler will be used.   Future versions will compile into machine code.  Then print out your FlowChart.

     

          Making and Modifying Symbols

 

This is done just like making a FlowChart, except that it works on the symbol screen where points, lines, and various texts are added, modified, or deleted.

  • Entering in the macro code text to create the named points and the syntax string.  Then just add the lines.

  • To add a line, select the color desired in the color drop down.

  • To add a point, select the data type from the data type drop down.

This lowest level symbol has the graphic picture for the symbol that is seen on the screen and in the printouts.  A symbol has syntax to create source code.  This syntax is nothing more than programmer code where the point names are replaced with the path names, so that one symbol can be used over and over again to create source code that is  unique for each symbol placed.  When macro code is entered, the syntax is automatically made.

 

 

Each symbol is one or two functions.

  • One is just another FlowChart because it's a subroutine or function in that computer language.

  • Two is the bottom level definition of the computer language.

 

Both of these functions have a symbol text that defines how the graphics are converted into text.  A FlowChart for a subroutine or function will have two parts:

  • One is a FlowChart that defines the routine being called, such as, 

    Function SubRoutine(X as integer, Y as real) as long

  • Two is a symbol that you place on flowcharts to call that routine, such as :

 Call SubRoutine(x, y) 

TIP:  Think of each symbol as doing something, whether it is just adding two numbers together or sorting a megabyte data file.  Think of the paths that go between the symbols as the data is flowing between symbols.

 

Some symbols will use data as input on the paths, others will set data as output on the paths, and others will do both by modifying the input at the start, and then putting it as output at the end.

 

Each symbol point will show as input, output, both or other options in the symbol  graphics as arrows showing the direction(s) of the logic or data flow.

Glossary

FlowChart : Is a graphical representation of the logic and data paths of a program connecting symbols together to show the logic steps of a computer process.

FlowChart graphics table : This refers to 'records' that hold the information about each located symbol, path segments, and constants.

Symbol : Is a graphical representation of a basic unit inside a FlowChart made up of lines and points. It contains lines, points, and other information to make the FlowChart into a program.

located symbol : This is a /use record in the FlowChart graphics table

Line : Is a graphical line in a symbol. It has color, but no width.

path segment :

Path : Is a graphical line in a FlowChart that represents the logic, or data going from one symbol to another. It has a color and width representing the type of data that that is moving in the program.

Point : Also called a connection point is in a symbol that is where the Paths from different symbols connect. It show where a path goes into, and comes out of a symbol. It has color and width for this type of data for this connection.

Connection Point: Is another name for Point.

Color : This is a name that Windows has a set of colors for. (Currently new color pens are not allowed, only the existing 139 color names.)

Data Type : This refers to the type of data that they symbol point is looking for. Some computer languages get very picky about this (Like C++), other computer languages could not care less (like BASIC).

DeCompiled : Thiss term refers to the process of converting a source code text file into the graphical FlowChart Computer Language.
